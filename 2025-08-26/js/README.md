브라우저에 직접 탑재될 수 있도록 가볍게 설계되었고, 별도의 컴파일 과정 없이 HTML 안에 <script> 태그로 바로 실행할 수 있게 만들어졌다.

자바스크립트의 가능성을 본 여러 회사들은 각자의 브라우저에 다양한 기능을 탑재하기 시작했고, 이로 인해 동일한 웹사이트가 브라우저마다 다르게 동작하는 크로스 브라우징 문제가 발생했다. 이러한 파편화는 자바스크립트 표준화의 필요성을 불러왔고, 그 결과 ECMAScript가 등장하게 되었다.

현대에 와서 자바스크립트는 브라우저 환경이 제공하는 웹 API와 ECMAScript 명세를 함께 의미한다.
반면 Node.js는 브라우저 밖의 실행 환경으로, 자체적으로 제공하는 API와 ECMAScript를 기반으로 동작한다.

2. 변수란 무엇인가? 변수 선언과 초기화 과정

예를 들어 10 + 10 과 같은 식이 있다.
이 연산을 수행하기 위해 컴퓨터는 숫자 10, 연산자 +, 그리고 전체 식을 해석할 수 있어야 한다. 인터프리터는 식을 해석하여 명령된 연산을 수행하는데, 이때 필요한 값(10, +, 10)을 메모리에 올려 참조한다. 연산이 끝나면 그 결과 20도 메모리에 저장된다.

하지만 단순히 메모리에만 저장된 값은 그 주소(위치)를 알 수 없기 때문에 이후 다시 접근하거나 활용하기 어렵다. 이 문제를 해결하기 위해 변수라는 개념이 도입되었다.

변수란 값을 담을 수 있는 메모리 공간을 가리키는 이름이다.
인터프리터가 var, let, const 같은 키워드를 만나면 새로운 변수를 선언하고, 내부적으로는 빈 공간을 찾아 해당 공간의 주소를 변수명과 연결해 둔다. 이렇게 하면 이후에는 변수명을 통해 그 공간에 저장된 값을 쉽게 참조할 수 있다.

선언 직후 별도의 값을 할당하지 않으면, 자바스크립트는 자동으로 undefined라는 값을 넣어 초기화한다. 이는 해당 메모리 공간에 남아 있을 수 있는 불필요한 쓰레기값(garbage data)을 안전하게 제거하고, 일관된 상태에서 변수를 사용할 수 있도록 보장하기 위함이다.

2. 매일메일 `closure`란 무엇인가?

```
클로저(Closure)란 함수가 선언될 당시의 환경(렉시컬 스코프)을 기억하여, 함수 실행이 끝난 이후에도 그 환경의 변수에 접근할 수 있는 것을 말한다.

function counter() {
  let count = 0;
  return () => {
    count++;
    return count;
  }
}

console.log(count); // ReferenceError
let increase = counter();
console.log(increase()); // 1

위 예시에서 count 변수는 counter 함수의 렉시컬 스코프에 존재하기 때문에 외부에서 직접 접근할 수 없다.
하지만 counter 함수가 반환하는 내부 함수(익명 함수)는 count를 참조하고 있으므로, increase 함수는 실행이 끝난 counter 함수의 내부 변수 count에 계속 접근할 수 있다. 이것이 바로 자바스크립트의 클로저다.

클로저는 외부에서 직접 접근할 수 없는 값을 안전하게 감추고, 오직 특정 함수 동작을 통해서만 제어할 수 있도록 할 때 유용하다.
```
